Здесь описан интерфейс таким, как я его вижу.

Окно вертикальное, не изменяется в соотношении и размерах. То есть кнопка maximize либо отсутсвует, либо не работает.

Теперь по самим меню, визуально как они должны выглядеть я привел в папке @refs. Я разработал дизайн.

Пиксельный шрифт это Bitcount Grid Double Regular, в основном используется 24 размер, только в контекстном меню размер его 18. На центральной кнопке на  меню используется этот же шрифт 56 размера.
Декоративный шрифт это Sekuya Regular, на entry это 80 размер, на всех остальных, где он есть - 56.
Шрифт для "ручного текста" на entry guide это Bad Script Regular 20 размера.


Цвета:
черный - 000000
белый - ffffff
зеленый - 57c086
красный - EA2F2F


Логика интерфейса в целом следуюущая: нас сначала встречает entry, на нем пользователь либо переходит по ссылке на нашего бота с подписками (сейчас там mock qr код), либо у него уже есть ссылка, и он идет дальше. 
Затем он перебрасывается на entry guide, на нем ему кратко расскажут, как включать, выключать и вставлять подключения, после нажатия на let me in! его перебрасывает на main меню.
На экранах entry и entry guide надписи, на которые наводятся мышь, через ховер подчеркиваются тем же цветом, что и текст. На entry guide кнопки off, /settings, /connections должны быть неактивными. 
На main меню слев наверху есть дата и время в реальном времени, подстраивается под устройство пользователя.
На этом экране есть 3 активные кнопки: центральная и 2 внизу: /settings, /connections.
В зависимости от того, в каком состоянии сейчас прокси, вид центральной кнпоки меняется. У него есть состояние off (см. main - off), состояние active (см. main - active), состояние error (см. main - error). В состоянии active от отсичтывает время активной сессии, а если error, пишет кратко тип ошибки: host problem, server problem, subscription problem. В режиме off там просто текст. 
На этом меню работает наш зовер только для кнопок /settings, /connections.
Если пользователь нажимает на кнопку /connections, его перебрасывает на connections - main. <- back to the main menu вовращает на main. Добавленные подключения по очереди заполняют поле и уходят сверзу вниз. Чтобы они не пересекались с кнопкой add connection, при приближении уходят в черный через градиент. На этом меню работает контекстное меню: при нажатии правой клавиши появляется черный прямоугольник с обводкой, можно увидеть на connections - context menu. На остальных меню контекстного меню нет! Ни кастомного, ни дефолтного не должно быть!
Если пользователь нажал на add connection, его перебрасывает на connections - add  меню. Дальше есть 2 версии: он может импортировать готовые vless ссылки или ссылку вида https://loki-panel.shmoza.net:8000/sub/SzlYMlI3TDVNNFE4WjFQQiwxNzc4MzMyNjQ3lGVgfDSRJi. При наведении Import link from clipboard или Import via subscription link они меняют цвет с черного на наш зеленый. Логика такая: пользователь нажимает add connection, его перебрасывает на connections - add, он со скопированной ссылкой нажимает Import link from clipboard или Import via subscription link, после чего его автоматически перебрасывает на connections - main. При этом подключение добавляется в список. 
settings - main меню отвечает за меню настроек. back to the main menu так же вовращает на main. пока пусть будет только Routing settings. Я не стал дальше расписывать цепочку настроек ,я просто прикреплю скрины из routing settings v2rayn, постарайся поддерживать наш стиль сам настроить эти меню.
